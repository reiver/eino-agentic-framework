RAG Reranking

document.Transformer score.NewReranker

by Charles Iliya Krempeaux

Retrieval-Augmented Generation (RAG) is a multi-step process. Two steps in RAG are:

  1. retrieval
  2. reranking

You can see how retrieval and reranking fit together in the RAG pipeline in the following figure:

With Retrieval-Augmented Generation (RAG), reranking is an important next step to be done after RAG retrieval. RAG reranking is an AI technique that improves the results returned from a Retrieval-Augmented Generation (RAG) retrieval.

The RAG retrieval step tends to be a quick. It uses bi-encoders to do vector search. This step might return 50 to 100 documents. The advatange is that it is fast. The disadvantage is that it misses the nuance of the prompt. So, for example, it doesn't really understand which of those documents are the best match with the prompt. (The best matches with the vector embedding are often not the best matches with the prompt.)

Where the RAG retrieval step is quick, the RAG reranking step can be slow. It uses cross-encoders to reorder the (for example) 50 to 100 documents returned from the RAG retrieval step. It (usually) looks at both the document and the prompt together (rather than just looking at vector embeddings). The advantage is that it orders the documents (in terms of best match) by paying attention to the prompt, rather than just the vector embedding of the prompt. The disadvantage is that it is slow, but since we are running it on (for example) only 50 to 100 documents (rather than millions of documents) it is practical to do.

  • RAG retrieval maximizes recall. (I.e., don't miss relevant documents).
  • RAG reranking maximizes precision. (I.e., it puts the best documents first).

With the Eino Agentic Framework, this is done with:

  • the Eino retriever.Retriever interface
  • the Eino document.Transformer inteface
  • the Eino score.NewReranker() func
