RAG Grounding

reduce hallucinations

by Charles Iliya Krempeaux

RAG grounding is he process of forcing a Large Language Model (LLM) to anchor its response strictly in the retrieved source documents, rather than relying on its pre-trained internal knowledge. It helps prevent hallucinations and ensure accuracy.

Grounding Strategies

Here are some grounding strategies:

Score Thresholding

Filtering out documents that fall below some relevance score.

Maximal Marginal Relevance (MMR)

Selecting documents/chunks that are relevant but not redundant (with some other document/chunk already selected).

Grounding Prompt (Guardrail)

You write the prompt in such a way to try to make the LLM draw an answer from the provided information. For example:

You are a helpful assistant. Answer the user's question only using the provided context below. If the answer cannot be found in the context, say "I do not know.". Do not make up information.
